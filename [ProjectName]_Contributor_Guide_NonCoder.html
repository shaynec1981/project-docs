This guide covers the universal rules for working in this repository — getting set up, branching, and submitting work. Sections 01–06 apply to everyone. Section 07 contains each department's own standards, owned by their respective lead. If you have questions, reach out in your team's Discord channel or directly to your lead.

Section 03

Workflow

Step 1
Create a branch
Branch → New Branch. See Section 04 for naming.
Step 2
Add your files
Place files in the correct folder. See Section 06.
Step 3
Commit
Write a short description of what you added. Click Commit.
Step 4
Push
Click Push origin to upload your branch.
Step 5
Open PR
Branch → Create Pull Request. Write a brief summary.
Step 6
Notify your lead
Message them on Discord — don't rely on GitHub notifications.
Never work directly on a department branch

Always create your own feature branch. Never push directly to art, sound, levdev, code, dev, or any other shared branch.

Section 05

Repo Rules

These rules apply to everyone regardless of department.

No spaces in file or folder names

ValidInvalid
forest_ambient_loop.wavforest ambient loop.wav
lake-fish-idle.pngLake Fish Idle.png

Don't touch .import files

Godot auto-generates a .import file alongside every asset. Never edit or delete them — just commit them as-is when they appear.

Commit messages

Write what you actually did. "Add idle sprites for jellyfish NPC" ✓    "stuff" ✗    "update" ✗

This is a simplified view of the repository. The full structure has more depth — if you're unsure where something belongs, ask in your Discord channel or check with your lead before placing it.

Off-limits — do not modify without permission

scripts/
Code only — contact Drykul
addons/
Auto-managed plugins — never touch

Accidentally changed something?

Don't commit it. In GitHub Desktop, right-click the file → Discard Changes. If already committed, contact Drykul on Discord immediately before pushing.

Section 07

Department Standards

Each department's standards are owned and maintained by their lead. Reach out to your lead or department Discord channel for questions.

Art ART Lead: LudicrousBiscuit · #artwork

Naming — general rules: snake_case · versions use two digits (01, 02) · names must be descriptive

Asset typePrefixPatternExample

Materialm_m_[category]_[name]_[version]m_pbr_jellytree_13

Workflow: Mechanics built by programmers → Prototype art → Testing → Adjust (repeat) → Polished art

Prototype art = kitbash assets or napkin sketches/stick figures. These are expected at early stages — they are not final.

Submission:

  • For review or discussion of an asset → post in #artwork tagged appropriately
  • For submissions → create a folder with your name in the Artwork Assets Drive on the shared Google Drive and post your work there
